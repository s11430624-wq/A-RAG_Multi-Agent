# M7-E.28 正式報告用精選表格

## 選表原則

本文件只保留適合放入正式報告的表格。

目前保存下來的歷史實驗紀錄中，包含多次中途中止、pilot 階段測試、以及不同穩定化階段的重跑結果。這些資料對工程審計很有價值，但若全部放進正文，會讓報告失焦。因此本文件只保留最有解釋力的四類表格：

1. **完整 45 筆實驗子集**：最適合拿來做主結果比較。
2. **完整覆蓋表**：證明 A / C / E 與 T01-T05 的配置在完整子集中是平衡的。
3. **歷史總摘要表**：用精簡方式交代整個 M7 full-run 過程的總工作量。
4. **失敗型態摘要表**：說明反覆 rerun 的主要原因，而不是逐次列舉所有 partial run。

下列所有數值皆於 2026-06-13 直接由目前磁碟上的 `results/raw/m7e_full_*.jsonl` 重新計算而得，而非沿用較早期的里程碑文字描述。

## 5. 建議在正式報告中的寫法

建議你在正文中這樣使用：

- **主結果比較**：使用「ACE 策略通過率表」作為主要量化結果。
- **覆蓋完整性說明**：使用「完整覆蓋表」證明完整子集沒有 task 或 strategy 偏斜。
- **過程摘要**：使用「歷史總摘要表」說明這不是一次就成功的單次 run，而是經過多輪穩定化後才形成的結果。
- **限制與失敗分析**：使用「失敗型態摘要表」說明主要瓶頸不是 infrastructure outage，而是 `repair_limit` 與 patch 套用不穩定。

## 6. 本文件刻意不放的內容

以下內容刻意不放進正式報告表格：

- 每個 experiment 一列的完整附錄流水帳；
- 所有只跑到 T01 或 T01-T02 的 partial run 逐筆列舉；
- 混合歷史全集的超細 task x strategy 大表。

這些資料仍然存在於原始 JSONL 中，適合做內部稽核，但不適合直接放入正式報告正文。
